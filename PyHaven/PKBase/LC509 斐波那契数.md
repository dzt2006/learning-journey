# 斐波那契数

### 题目描述

斐波那契数（通常用 F(n) 表示）形成的序列称为斐波那契数列。该数列由 0 和 1 开始，后面的每一项数字都是前面两项数字的和。也就是：

- F(0) = 0，F(1) = 1
- F(n) = F(n - 1) + F(n - 2)，其中 n > 1

给定 n，请计算 F(n)。

**示例 1：**

输入：`n = 2`
输出：`1`
解释：F(2) = F(1) + F(0) = 1 + 0 = 1

**示例 2：**

输入：`n = 3`
输出：`2`
解释：F(3) = F(2) + F(1) = 1 + 1 = 2

**示例 3：**

输入：`n = 4`
输出：`3`
解释：F(4) = F(3) + F(2) = 2 + 1 = 3

**提示：**

- 0 <= n <= 30

***

### 解法一：动态规划法

斐波那契数列的递推公式已经直接给了：F(n) = F(n-1) + F(n-2)。所以我们只需要从 F(0) 和 F(1) 开始，一步步往后算，直到算出 F(n) 就行。

我们用一个数组 `dp` 来存每个位置的值，`dp[i]` 就代表 F(i)。

```python
def fib(n: int) -> int:
    if n == 0:
        return 0
    if n == 1:
        return 1

    dp = [0] * (n + 1)

    dp[0], dp[1] = 0, 1
    for i in range(2, n + 1):
        dp[i] = dp[i - 1] + dp[i - 2]

    return dp[n]
```

#### 逐步调试

用示例 3 来走一遍：`n = 4`

**初始化：**

`dp = [0, 1, 0, 0, 0]`（长度为 5，dp[0]=0, dp[1]=1）

**第 1 轮：i=2**

- dp[2] = dp[1] + dp[0] = 1 + 0 = 1
- dp = [0, 1, 1, 0, 0]

**第 2 轮：i=3**

- dp[3] = dp[2] + dp[1] = 1 + 1 = 2
- dp = [0, 1, 1, 2, 0]

**第 3 轮：i=4**

- dp[4] = dp[3] + dp[2] = 2 + 1 = 3
- dp = [0, 1, 1, 2, 3]

返回 dp[4] = 3 ✓

**测试：**

```python
print(f"n=2: {fib(2)}")
print(f"n=3: {fib(3)}")
print(f"n=4: {fib(4)}")
```

**执行结果：**
```
n=2: 1
n=3: 2
n=4: 3
```

#### 时间复杂度分析

时间复杂度是 **O(n)**，只需要遍历一次。空间复杂度是 **O(n)**，因为用了一个长度为 n+1 的数组。

***

### 解法二：空间优化法

仔细想想，算 F(n) 的时候其实只需要前两个值 F(n-1) 和 F(n-2)，更早的值根本用不到了。那我们何必存整个数组呢？用两个变量就够了！

```python
def fib1(n: int) -> int:
    if n < 2:
        return n

    a, b = 0, 1
    for _ in range(2, n + 1):
        a, b = b, a + b

    return b
```

#### 逐步调试

用示例 3 来走一遍：`n = 4`

**初始状态：** a=0, b=1

**第 1 轮：**

- a, b = b, a+b → a=1, b=0+1=1
- 此时 a=F(1)=1, b=F(2)=1

**第 2 轮：**

- a, b = b, a+b → a=1, b=1+1=2
- 此时 a=F(2)=1, b=F(3)=2

**第 3 轮：**

- a, b = b, a+b → a=2, b=1+2=3
- 此时 a=F(3)=2, b=F(4)=3

返回 b = 3 ✓

**测试：**

```python
print(f"n=2: {fib1(2)}")
print(f"n=3: {fib1(3)}")
print(f"n=4: {fib1(4)}")
```

**执行结果：**
```
n=2: 1
n=3: 2
n=4: 3
```

#### 时间复杂度分析

时间复杂度还是 **O(n)**，但空间复杂度降到了 **O(1)**，因为我们只用了两个变量，不再需要额外的数组。

***

### 两种解法对比

| 解法 | 时间复杂度 | 空间复杂度 | 特点 |
|------|-----------|-----------|------|
| 动态规划法 | O(n) | O(n) | 直观易懂，用数组存所有中间结果 |
| 空间优化法 | O(n) | O(1) | 只保留前两个值，省空间 |

空间优化法是动态规划的常见优化思路：如果当前状态只依赖前面有限个状态，就不需要存整个数组，只保留需要的几个变量就够了。

# 两数之和

### 题目描述

给定一个整数数组 `nums` 和一个整数目标值 `target`，请你在该数组中找出和为目标值 `target` 的那两个整数，并返回它们的数组下标。

你可以假设每种输入只会对应一个答案，并且你不能使用两次相同的元素。你可以按任意顺序返回答案。

**示例 1：**

输入：`nums = [2, 7, 11, 15]`, `target = 9`
输出：`[0, 1]`
解释：因为 nums[0] + nums[1] == 9，返回 [0, 1]

**示例 2：**

输入：`nums = [3, 2, 4]`, `target = 6`
输出：`[1, 2]`

**示例 3：**

输入：`nums = [3, 3]`, `target = 6`
输出：`[0, 1]`

***

### 解法一：暴力枚举法

最直观的思路：用两层循环，把所有两两组合都试一遍，找到和等于 target 的那一对。

外层循环遍历每个元素 `x`，内层循环从 `x` 后面开始遍历每个元素，如果 `x + nums[j] == target`，就返回它们的下标。

```python
from typing import List

def twoSum_brute(nums: List[int], target: int) -> List[int]:
    for i, x in enumerate(nums):
        for j in range(i + 1, len(nums)):
            if x + nums[j] == target:
                return [i, j]
    return []
```

**测试：**

```python
print("示例1 - 暴力枚举法:", twoSum_brute([2, 7, 11, 15], 9))
print("示例2 - 暴力枚举法:", twoSum_brute([3, 2, 4], 6))
print("示例3 - 暴力枚举法:", twoSum_brute([3, 3], 6))
```

**执行结果：**
```
示例1 - 暴力枚举法: [0, 1]
示例2 - 暴力枚举法: [1, 2]
示例3 - 暴力枚举法: [0, 1]
```

#### 时间复杂度分析

两层循环，最坏情况下要遍历 n×(n-1)/2 次，时间复杂度是 **O(n²)**。

***

### 解法二：哈希表法

暴力枚举法太慢了，能不能只遍历一次就找到答案？可以！用哈希表（字典）来帮忙。

思路是这样的：遍历数组时，对于当前元素 `val`，我们想知道"之前有没有出现过另一个数，使得它和 `val` 的和等于 target"。换句话说，我们之前有没有见过 `target - val` 这个数？

所以我们用一个字典，把 `target - val` 作为键、当前索引作为值存进去。每遍历到一个新元素，先查字典里有没有它，有就直接返回答案，没有就继续存。

```python
def twoSum_hash(nums: List[int], target: int) -> List[int]:
    a = {}
    for i, val in enumerate(nums):
        if val in a:
            return [a[val], i]
        a[target - val] = i
    return []
```

#### 逐步调试

用示例 2 来走一遍：`nums = [3, 2, 4]`, `target = 6`

**第 1 轮：i=0, val=3**

- 查字典：3 在 a 里吗？→ 不在，a 是空的
- 存入字典：`a[6-3] = 0`，即 `a[3] = 0`
- 当前字典：`{3: 0}`（意思是：如果后面遇到 3，就知道索引 0 这个数在等它）

**第 2 轮：i=1, val=2**

- 查字典：2 在 a 里吗？→ 不在
- 存入字典：`a[6-2] = 1`，即 `a[4] = 1`
- 当前字典：`{3: 0, 4: 1}`（索引 0 等 3，索引 1 等 4）

**第 3 轮：i=2, val=4**

- 查字典：4 在 a 里吗？→ **在！** `a[4] = 1`
- 找到了！返回 `[a[4], 2]`，即 `[1, 2]`
- 索引 1 的值是 2，索引 2 的值是 4，2 + 4 = 6 ✓

**测试：**

```python
print("示例1 - 哈希表法:", twoSum_hash([2, 7, 11, 15], 9))
print("示例2 - 哈希表法:", twoSum_hash([3, 2, 4], 6))
print("示例3 - 哈希表法:", twoSum_hash([3, 3], 6))
```

**执行结果：**
```
示例1 - 哈希表法: [0, 1]
示例2 - 哈希表法: [1, 2]
示例3 - 哈希表法: [0, 1]
```

#### 时间复杂度分析

只需要遍历一次数组，每次查字典是 O(1)，所以总时间复杂度是 **O(n)**。空间复杂度是 O(n)，因为最坏情况下要存 n 个键值对。

***

### 两种解法对比

| 解法 | 时间复杂度 | 空间复杂度 | 特点 |
|------|-----------|-----------|------|
| 暴力枚举法 | O(n²) | O(1) | 简单直观，但慢 |
| 哈希表法 | O(n) | O(n) | 用空间换时间，快 |

哈希表法用额外的空间换取了更快的时间，这是算法中常见的"空间换时间"策略。

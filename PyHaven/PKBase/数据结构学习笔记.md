## 什么是数组

数组是一种线性数据结构，它将相同类型的元素存储在连续的内存空间中。数组的特点包括：

- 连续的内存空间
- 相同类型的元素
- 可通过索引快速访问元素（O(1) 时间复杂度）

## 随机访问元素

知道了什么是数组，那么我们就可以写一个实现 O(1) 时间复杂度的元素随机访问啦！

### 随机访问原理

数组在内存中是连续存储的，通过索引可以直接计算出元素的内存地址，从而实现 O(1) 时间复杂度的访问。

```python
def random_access(nums: list[int]) -> int:
    # 生成一个随机索引
    random_index = random.randint(0, len(nums) - 1)
    # 通过随机索引访问元素
    random_nums = nums[random_index]
    return random_nums

# 测试随机访问
print("随机访问的元素:", random_access(nums))
```

## 删除元素

现在就是数组的一些简单的操作了，首先就是删除。

### 删除原理

删除元素后，原本末尾的元素就丢失了，没有意义了，就没有必要去修改它。

当删除索引 index 处的元素时，我们需要从 index 开始遍历，把删除位置后面的所有元素，依次往前移动一位。

```python
def remove(nums: list[int], index: int):
    # 检查索引是否有效
    if index < 0 or index >= len(nums):
        print("索引无效")
        return
    
    # 从 index 开始，后面的元素依次往前移动
    for i in range(index, len(nums) - 1):
        nums[i] = nums[i + 1]

# 测试删除操作
print("删除前:", nums)
remove(nums, 2)  # 删除索引为 2 的元素（37）
print("删除后:", nums)
```

### 时间复杂度分析

删除操作的时间复杂度是 O(n)，因为我们需要移动被删除位置后面的所有元素。

## 查找元素

接下来就是查找元素。

### 查找实现

在数组中查找指定元素，返回该元素的索引。如果没有找到，返回 -1。

```python
def find(nums: list[int], target: int) -> int:
    # 遍历数组，查找目标元素
    for i in range(len(nums)):
        if nums[i] == target:
            return i  # 找到目标，返回索引
    return -1  # 没找到，返回 -1

# 测试查找
target = 47
result = find(nums, target)
if result != -1:
    print(f"找到 {target}，索引位置为 {result}")
else:
    print(f"未找到 {target}")
```

### 时间复杂度分析

查找操作的时间复杂度是 O(n)，因为在最坏情况下我们需要遍历整个数组。

## 数组扩容

到这一步，聪明的你肯定会发现：数组的长度是不可变的。那我们该如何对它进行扩容呢？

### 扩容原理

你一定会想到——创建一个容量更大的新数组，再把原数组的元素全部复制过去。但要注意，这个操作的时间复杂度是 O(n)，在数组规模非常大时，会非常耗时。

```python
# enlarge：额外扩容的个数
def extend(nums: list[int], enlarge: int) -> list[int]:
    # 创建新数组，长度为原数组长度 + 扩容个数
    res = [0] * (len(nums) + enlarge)
    
    # 将原数组的元素复制到新数组
    for i in range(len(nums)):
        res[i] = nums[i]
    
    return res

# 测试扩容
print("原数组:", nums)
new_nums = extend(nums, 3)  # 扩容 3 个位置
print("扩容后:", new_nums)
```

## 什么是链表

链表也是一种线性数据结构，但和数组不一样，链表的元素在内存中不需要连续存放。它通过"指针"（引用）把一个个节点串起来，就像一条链子一样。

链表的特点包括：

- 内存不连续，每个节点存自己的值和下一个节点的位置
- 插入和删除效率高，不用像数组那样移动元素（O(1) 时间复杂度）
- 访问元素只能从头开始一个一个找，不能像数组那样通过索引直接定位（O(n) 时间复杂度）

***

## 链表节点定义

链表的基本单位是节点（ListNode），每个节点包含两部分：存储的值（val）和指向下一个节点的引用（next）。

```python
from typing import Optional

# 只有当这个参数 / 变量真的有可能是 None 的时候，才需要加 Optional
class ListNode:
    def __init__(self, val: int = 0, next: Optional["ListNode"] = None):
        self.val: int = val
        self.next: Optional[ListNode] = next
```

***

## 链表初始化

初始化链表分两步：先创建各个节点，再构建节点之间的引用关系。

```python
# 初始化链表 1 -> 3 -> 2 -> 5 -> 4

# 第一步：创建各个节点
n0 = ListNode(1)
n1 = ListNode(3)
n2 = ListNode(2)
n3 = ListNode(5)
n4 = ListNode(4)

# 第二步：构建节点之间的引用
n0.next = n1
n1.next = n2
n2.next = n3
n3.next = n4
```

***

## 插入节点

在链表中插入节点非常方便，只需要修改指针的指向就行，不用像数组那样移动后面的所有元素。

### 插入原理

假设要在节点 n 后面插入新节点 P，操作分三步：

1. 把 n 原本指向的下一个节点先"剥离"出来
2. 让新节点 P 指向这个剥离出来的节点
3. 再让 n 重新指向新节点 P

```python
def insert(n: ListNode, P: ListNode):
    # 把该节点原本的指向直接剥离出来
    n0 = n.next
    # 把剥离出来的这个指向，直接赋给新加入节点的指向
    P.next = n0
    # 再让原本剥离指向的节点，重新指向新节点
    n.next = P
```

**示例：**

```python
print("原始链表：", end="")
print_List(n0)

insert(n1, ListNode(56))
print("插入后链表：", end="")
print_List(n0)
```

**执行结果：**
```
原始链表：1 -> 3 -> 2 -> 5 -> 4
插入后链表：1 -> 3 -> 56 -> 2 -> 5 -> 4
```

### 时间复杂度分析

插入操作的时间复杂度是 O(1)，只需要修改两个指针的指向。

***

## 删除节点

删除节点和插入类似，也是修改指针的指向，跳过要删除的节点就行。

### 删除原理

假设要删除节点 n 后面的节点，操作分两步：

1. 找到要删除的节点 P 和它后面的节点 n0
2. 直接让当前节点 n 指向 n0，跳过 P 就完成了删除

```python
def remove(n: ListNode):
    if not n.next:
        return
    # n 是当前节点，P 是要删除的节点，n0 是要删除节点后面的节点
    P = n.next
    n0 = P.next
    # 直接让当前节点指向删除节点后面的节点就可以了
    n.next = n0
```

**示例：**

```python
print("删除前：", end="")
print_List(n0)

remove(n1)  # 删除 n1(值为3) 后面的节点
print("删除后：", end="")
print_List(n0)
```

**执行结果：**
```
删除前：1 -> 3 -> 56 -> 2 -> 5 -> 4
删除后：1 -> 3 -> 2 -> 5 -> 4
```

### 时间复杂度分析

删除操作的时间复杂度是 O(1)，只需要修改一个指针的指向。

***

## 访问节点

链表不能像数组那样通过索引直接访问，只能从头节点开始，顺着 next 一个一个走，走到目标位置为止。

```python
def access(head: Optional[ListNode], index: int) -> Optional[ListNode]:
    for i in range(index):
        if not head:
            return None
        head = head.next
    return head
```

**示例：**

```python
# 索引从 0 开始，0-1, 1-3, 2-2, 3-5
node = access(n0, 3)
if node:
    print(f"索引 3 的节点值：{node.val}")
else:
    print("索引超出范围")

node2 = access(n0, 10)
print(f"索引 10 的节点：{node2}")
```

**执行结果：**
```
索引 3 的节点值：5
索引 10 的节点：None
```

### 时间复杂度分析

访问操作的时间复杂度是 O(n)，因为需要从头遍历到目标位置。

***

## 查找节点

查找就是从头开始遍历，找到值等于目标的节点，返回它的索引。如果遍历完了还没找到，就返回 -1。

```python
def find(head: Optional[ListNode], target: int) -> int:
    index = 0
    while head:
        if head.val == target:
            return index
        head = head.next
        index += 1
    return -1
```

**示例：**

```python
pos1 = find(n0, 5)
pos2 = find(n0, 100)
print(f"值 5 的索引：{pos1}")
print(f"值 100 的索引：{pos2}")
```

**执行结果：**
```
值 5 的索引：3
值 100 的索引：-1
```

### 时间复杂度分析

查找操作的时间复杂度是 O(n)，最坏情况下需要遍历整个链表。

***

## 遍历链表

遍历链表就是从头节点开始，顺着 next 一直走，直到走完为止。

这里要注意一个细节：遍历时要用一个临时变量 `curr` 来代替 `head`，而不是直接用 `head`。因为如果直接用 `head` 遍历，执行 `head = head.next` 会不断移动头指针，等遍历结束，头指针就跑到链表末尾了，后续就找不到起始入口，无法再次访问完整的链表了。

```python
def print_List(head: Optional[ListNode]):
    # curr 是一个临时指针（游标），保护原始头指针不被修改
    curr = head
    while curr:
        print(curr.val, end=" ")
        if curr.next:
            print("->", end="")
        curr = curr.next
    print()
```

**示例：**

```python
print_List(n0)
```

**执行结果：**
```
1 -> 3 -> 2 -> 5 -> 4
```

## 什么是列表

列表其实就是"动态数组"，它和数组的区别就在于：数组的长度是固定的，而列表的长度可以随时变化。列表在底层还是用数组存的，但当空间不够的时候，它会自动扩容，不需要我们手动处理。

列表的特点包括：

- 长度可变，支持动态扩容
- 支持在任意位置插入和删除元素
- 底层还是连续的内存空间，访问效率高

## 列表的实现

Python 自带的列表虽然好用，但了解它底层是怎么实现的，能帮助我们更好地理解它的工作原理。下面我们自己动手实现一个简化版的列表！

### MyList 类定义

我们的 MyList 需要这几个核心属性：

- `_capacity`：列表的容量，也就是底层数组最多能放多少元素
- `_arr`：底层数组，用来实际存储元素
- `_size`：列表当前有多少个有效元素
- `_extend_ratio`：每次扩容的倍数

```python
class MyList:
    def __init__(self):
        self._capacity: int = 10
        self._arr: list[int] = [0] * self._capacity
        self._size: int = 0
        self._extend_ratio: int = 2
```

---

### 获取列表长度和容量

`size()` 返回当前有多少个有效元素，`capacity()` 返回底层数组的容量。

```python
def size(self) -> int:
    return self._size

def capacity(self) -> int:
    return self._capacity
```

---

### 扩容机制

当元素数量超过容量时，列表需要扩容。原理很简单：在原数组后面追加一段新空间，容量就变大了。这里我们按倍数扩容，比如 2 倍。

```python
def extend_capacity(self):
    self._arr = self._arr + [0] * self._capacity * (self._extend_ratio - 1)
    self._capacity = len(self._arr)
```

---

### 访问元素

通过索引直接访问底层数组中的元素。注意要先检查索引是否越界。

```python
def get(self, index: int) -> int:
    if index < 0 or index >= self._size:
        raise IndexError("索引越界")
    return self._arr[index]
```

---

### 更新元素

把指定索引位置的元素改成新值，同样要先检查索引是否越界。

```python
def set(self, num: int, index: int):
    if index < 0 or index >= self._size:
        raise IndexError("索引越界")
    self._arr[index] = num
```

---

### 在尾部添加元素

往列表末尾添加元素时，先检查容量够不够，不够就先扩容，然后把元素放到 `_size` 的位置，再把 `_size` 加 1。

```python
def add(self, num: int):
    if self._size == self._capacity:
        # 为什么偏要写 self 呢？因为 extend_capacity() 写在 class 里面，只有这样才能找到
        self.extend_capacity()
    self._arr[self._size] = num
    self._size += 1
```

### 时间复杂度分析

尾部添加的平均时间复杂度是 O(1)。最坏情况（触发扩容）是 O(n)，但扩容是偶尔才发生的，平摊下来每次添加还是 O(1)。

---

### 在中间插入元素

在指定索引位置插入元素，需要先把该位置及后面的元素都往后移一位，腾出空位，再放新元素进去。

```python
def insert(self, nums: int, index: int):
    if index < 0 or index >= self._size:
        raise IndexError("索引越界")

    # 元素数量超过容量时，触发扩容函数
    if self._size == self._capacity:
        self.extend_capacity()

    # 从后往前，把元素逐个往后移一位
    for i in range(self._size - 1, index - 1, -1):
        self._arr[i + 1] = self._arr[i]
    self._arr[index] = nums
    self._size += 1
```

在插入操作中，只需要移动目标索引位置及其之后的元素，索引之前的元素无需任何处理，前面的元素的索引直接不考虑了，这是一个很好的习惯。

### 时间复杂度分析

插入操作的时间复杂度是 O(n)，因为最坏情况下需要移动几乎所有元素。

---

### 删除元素

删除指定索引的元素，需要把该位置后面的元素都往前移一位，覆盖掉被删除的元素。

```python
def remove(self, index: int) -> int:
    if index < 0 or index >= self._size:
        raise IndexError("索引越界")
    num = self._arr[index]
    # 从前往后，把元素逐个往前移一位
    for i in range(index, self._size - 1):
        self._arr[i] = self._arr[i + 1]
    self._size -= 1
    return num
```

### 时间复杂度分析

删除操作的时间复杂度是 O(n)，和插入一样，最坏情况下需要移动几乎所有元素。

---

### 返回有效长度的列表

底层数组可能有很多空位，我们只返回有实际元素的那部分。

```python
def to_array(self) -> list[int]:
    return self._arr[: self._size]
```

## 什么是栈

栈是一种"后进先出"（LIFO, Last In First Out）的数据结构。你可以把它想象成一摞盘子——最后放上去的盘子，最先被拿走。

栈的特点包括：

- 只能从栈顶操作元素（入栈 push、出栈 pop）
- 最后入栈的元素最先出栈（后进先出）
- 访问栈顶元素的时间复杂度是 O(1)

Python 没有内置的栈类，但我们可以把 list 当作栈来使用，也可以自己基于链表或数组来实现。

***

## 基于列表的栈操作

Python 的 list 本身就提供了 `append()` 和 `pop()` 方法，天然适合当栈用。

```python
# 初始化栈
stack: list[int] = []

# 元素入栈
stack.append(1)
stack.append(3)
stack.append(2)
stack.append(5)
stack.append(4)

# 访问栈顶元素
peek: int = stack[-1]

# 元素出栈
pop: int = stack.pop()

# 获取栈的长度
size: int = len(stack)

# 判断是否为空
is_empty: bool = len(stack) == 0
```

虽然用 list 很方便，但了解栈的底层实现原理也很重要，下面我们分别用链表和数组来实现栈。

***

## 基于链表实现的栈

用链表实现栈，我们需要维护一个栈顶指针 `_peek` 和栈的长度 `_size`。入栈就是在头部插入新节点，出栈就是删除头节点。

### 节点定义

和链表一样，栈的节点也包含值和指向下一个节点的引用。

```python
from typing import Optional

class ListNode:
    def __init__(self, val: int = 0, next: Optional["ListNode"] = None):
        self.val: int = val
        self.next: Optional[ListNode] = next
```

---

### LinkedListStack 类定义

```python
class LinkedListStack:
    def __init__(self):
        self._peek: Optional[ListNode] = None
        self._size: int = 0
```

---

### 获取栈的长度和判空

```python
def size(self) -> int:
    return self._size

def is_empty(self) -> bool:
    return self._size == 0
```

---

### 入栈

入栈就是创建一个新节点，让它指向原来的栈顶，然后更新栈顶指针。

```python
def push(self, val: int):
    # 创建一个新的节点
    node = ListNode(val)
    # 把新节点指向原来的栈顶节点
    node.next = self._peek
    # 更新栈顶指针为新节点
    self._peek = node
    self._size += 1
```

---

### 访问栈顶元素

直接返回栈顶节点的值就行，但要先判断栈是否为空。

```python
def peek(self) -> int:
    if self.is_empty():
        raise IndexError("栈为空")
    return self._peek.val
```

---

### 出栈

出栈就是返回栈顶的值，然后把栈顶指针往下移一位。

```python
def pop(self) -> int:
    num = self.peek()
    self._peek = self._peek.next
    self._size -= 1
    return num
```

---

### 打印栈（转为列表）

为了方便查看栈的内容，我们把链表从头到尾遍历一遍，转成列表输出。注意因为栈顶是链表头，所以需要反转一下顺序。

```python
def to_list(self) -> list[int]:
    arr = []
    node = self._peek
    while node:
        arr.append(node.val)
        node = node.next
    arr.reverse()
    return arr
```

---

### 完整示例

```python
# 1. 初始化栈
stack = LinkedListStack()
print(f"初始化后是否为空: {stack.is_empty()}")
print(f"初始化后栈长度: {stack.size()}")

# 2. 元素入栈
stack.push(1)
stack.push(3)
stack.push(2)
stack.push(5)
stack.push(4)
print(f"\n入栈 [1,3,2,5,4] 后栈: {stack.to_list()}")
print(f"入栈后栈长度: {stack.size()}")
print(f"当前栈顶元素: {stack.peek()}")

# 3. 元素出栈
pop_val = stack.pop()
print(f"\n出栈元素: {pop_val}")
print(f"出栈后栈: {stack.to_list()}")
print(f"出栈后栈长度: {stack.size()}")
print(f"出栈后栈顶元素: {stack.peek()}")

# 4. 连续出栈直到为空
print("\n连续出栈:")
while not stack.is_empty():
    print(f"弹出: {stack.pop()}, 剩余栈: {stack.to_list()}")

# 5. 空栈操作测试
try:
    stack.pop()
except IndexError as e:
    print(f"\n空栈pop()抛出异常: {e}")
```

**执行结果：**
```
初始化后是否为空: True
初始化后栈长度: 0

入栈 [1,3,2,5,4] 后栈: [1, 3, 2, 5, 4]
入栈后栈长度: 5
当前栈顶元素: 4

出栈元素: 4
出栈后栈: [1, 3, 2, 5]
出栈后栈长度: 4
出栈后栈顶元素: 5

连续出栈:
弹出: 5, 剩余栈: [1, 3, 2]
弹出: 2, 剩余栈: [1, 3]
弹出: 3, 剩余栈: [1]
弹出: 1, 剩余栈: []

空栈pop()抛出异常: 栈为空
```

## 基于数组实现的栈

用数组实现栈更直观——数组的末尾就是栈顶，`append()` 就是入栈，`pop()` 就是出栈。

### ArrayStack 类定义

```python
class ArrayStack:
    def __init__(self):
        self._stack: list[int] = []
```

---

### 获取栈的长度和判空

```python
def size(self) -> int:
    return len(self._stack)

def is_empty(self) -> bool:
    return self.size() == 0
```

---

### 入栈

直接在数组末尾追加元素就行。

```python
def push(self, item: int):
    self._stack.append(item)
```

---

### 出栈

弹出数组末尾的元素，先判断栈是否为空。

```python
def pop(self) -> int:
    if self.is_empty():
        raise IndexError("栈为空")
    return self._stack.pop()
```

---

### 访问栈顶元素

返回数组最后一个元素就是栈顶。

```python
def peek(self) -> int:
    if self.is_empty():
        raise IndexError("栈为空")
    return self._stack[-1]
```

---

### 打印栈（转为列表）

数组栈的内部存储顺序和栈的逻辑顺序一致，直接返回就行。

```python
def to_list(self) -> list[int]:
    return self._stack
```

---
